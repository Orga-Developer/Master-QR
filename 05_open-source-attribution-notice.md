## Purpose
This notice provides a concise but complete attribution framework for open-source components distributed with the application. It serves as the public-facing record that preserves authorship, copyright notices, and license references required by upstream maintainers.

## Scope
It applies to the app bundle, downloadable assets, documentation exports, and any store-supplied source archive that includes third-party code. The notice should be readable from the app, from a companion website, or from an archived legal bundle shipped with the release.

## Operational Commitments
Attribution must include the component name, the license family, and any mandatory disclaimer text or upstream copyright statement. The notice should not overclaim ownership and should avoid burying required acknowledgements inside unrelated prose. Where multiple components share the same license, the record should still identify each component clearly.

## Application to Master QR
For Master QR, attribution is especially relevant because the build uses a modern Android stack with barcode, camera, Compose, Room, and security tooling. The notice should be synchronized with the exact versions shipped in the release and with the combined legal pack exposed in the app’s assets.

## Review and Governance
The attribution file should be regenerated whenever the dependency graph changes. Automated checks are recommended so that omitted or stale notices are caught before store submission.
