# URL and Link Handling Policy

Whenever a decoded QR code contains a URL, the application must manage the link in a way that is legible, reversible, and safe by default. The user should see the destination clearly before any external browser or app intent is launched. The policy requires domain display, scheme awareness, and refusal of suspicious or malformed destinations whenever feasible. Deep links, redirects, and shortened URLs should be handled cautiously because they can conceal the ultimate target. The application may allow copy-to-clipboard and share operations, but those features should still preserve user awareness of the source and destination. If link previews, safety checks, or external intent routing are added, their logic must be documented and tested in release candidates.
