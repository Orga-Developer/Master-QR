# Sensitive Configuration Management Policy

Sensitive configuration values such as API keys, ad IDs, feature toggles, environment-specific endpoints, and consent configuration metadata must be managed as controlled operational assets. Master QR should treat these values differently from ordinary source code because they can alter privacy posture, monetization behavior, or store compliance. The policy requires environment separation, access limitation, review trails, and documented rollback procedures for any configuration change affecting user data or partner integrations. Sensitive configuration should not be edited ad hoc in release emergencies without traceability. This discipline helps the team ensure that the application behaves consistently with the privacy policy and that production builds cannot quietly diverge from their approved disclosures.
