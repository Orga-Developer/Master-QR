# Firebase Data Flow Disclosure

The current codebase contains conditional Firebase initialization, which means data processing must be described precisely and conservatively. Firebase services, if enabled in a production variant, may support analytics, crash diagnostics, or remote configuration. Each service must be mapped to a specific business purpose, a storage location, a retention horizon, and a lawful basis where required. The application should not imply that Firebase is always active if the build can also operate without valid non-dummy credentials. Any Firebase-backed flow must be presented to users through a consistent disclosure stack that explains what is collected, why it is collected, and how the user may exercise opt-out or deletion rights where available under applicable law.
