# Data Selling Disclosure

This instrument formalizes the rules for Data Selling Disclosure and aligns them with the actual behavior of the application. Master QR is a camera-first Android application that scans QR codes, generates QR codes, stores scan history locally in Room, supports language switching, provides share/export flows, integrates advertising and analytics components, and is distributed across mobile app stores with platform-specific compliance requirements. The document is written in advanced legal English so it can serve as a publication-ready internal policy, a store-review support artifact, and a future contract annex. It is intentionally precise about scope, permissions, data categories, and release governance, so the text can survive scrutiny across multiple jurisdictions and app ecosystems.

Applicability to Master QR: This policy should clearly state whether the application sells data, and if not, should say so in explicit operational terms. In a QR-focused app, the safest legal posture is that user content, history, and device data are not sold as a product. If any monetization ecosystem is later expanded, the disclosure must distinguish between selling, sharing, processor access, and contextual ad delivery. This distinction matters because legal definitions differ across jurisdictions, and users must understand whether data is being commercialized or merely processed for service delivery. The policy must therefore distinguish between what the application actually does today and what it merely might do in a future roadmap. In the current codebase, that means recognizing camera-based scanning, QR generation, local history management, language preferences, optional exporting, advertising or analytics integrations, and the absence of broad account-based processing.

Operational requirements: the developer should document purpose limitation, legal basis where relevant, retention limits, access restrictions, vendor controls, user notice requirements, and escalation paths for exceptions. If a feature is not present, the document should not pretend that it is; instead it should state that the control is reserved for future expansion. That approach is especially valuable for store review, compliance review, and internal audit preparation.
